# 기본 보고서 양식: 4열 테이블

<!-- This file defines the default Korean report format. Keep content in Korean — it is the output template. -->

월간 피드백 보고서의 기본 양식. 사용자가 별도 양식을 제공하지 않으면 이 형식을 사용한다.

## 양식

```markdown
# YYYY년 MM월 업무 피드백

| 목표 | 핵심결과 | 잘한 점 | 부족한 점 및 보완계획 |
|------|---------|--------|-------------------|
| [서비스명] 상위 목표 요약 | • 구체적 구현 내용 1 • 구체적 구현 내용 2 | • (카테고리) 성과 설명 | • 부족한 점 설명 • 보완계획: 구체적 액션 |
```

## 각 열 작성 가이드

### 목표
- 서비스명을 대괄호로 표시: `[서비스명]`
- 개별 커밋이 아닌 상위 목적을 작성
- 예: "사용자 인증 플로우 개선 및 UI 고도화"

### 핵심결과
- 실제로 구현/완료한 것을 구체적으로 나열
- 기술적 키워드를 포함 (페이지명, 컴포넌트명, API명 등)
- 글머리 기호(•)로 구분
- 예: "• 로그인 페이지 리뉴얼 및 소셜 로그인 연동 구현"

### 잘한 점
- 괄호로 카테고리를 표시: `(생산성 향상)`, `(전반적인 품질 향상)`, `(디테일 챙기기)`
- 단순히 "잘했다"가 아니라 왜 잘한 것인지 설명
- 예: "• (생산성 향상) Storybook 전역 다국어 세팅 및 AI 에이전트 가이드라인을 선제적으로 구축해 개발 생산성 기반을 다짐"

### 부족한 점 및 보완계획
- 부족한 점과 보완계획을 함께 작성
- 보완계획은 구체적인 다음 액션으로 작성
- 예: "• 2단계 인증에서 실제 테스트를 수동으로 검수함 • 보완계획: 테스트 시나리오 및 자동 e2e 테스트가 필요해보임"

## 서비스별 행 구분

서비스가 여러 개면 행을 나눈다:

```markdown
| 목표 | 핵심결과 | 잘한 점 | 부족한 점 및 보완계획 |
|------|---------|--------|-------------------|
| **[서비스A]** 사용자 인증 플로우 개선 | • 로그인 페이지 리뉴얼 ... | • (생산성 향상) ... | • 테스트 수동 검수 ... |
| **[서비스B]** 검색 기능 고도화 | • 필터링 UI 개선 ... | • (품질 향상) ... | • 테스트 강화 필요 ... |
```

## 업무 리스트 양식

업무 리스트는 별도 문서로 생성한다:

```markdown
# YYYY년 MM월 업무 리스트

## [서비스명] 레포명 (N일 작업, M커밋)

1. ✅ 완료된 태스크 제목
   - 기간: M/D ~ M/D (N일)
   - 커밋 N건, MR !번호 머지
   - 주요 결정: 간단한 설명

2. 🔧 진행 중인 태스크 제목
   - 기간: M/D ~ (진행중)
   - 커밋 N건

## 코드 외 업무
- 코드리뷰: N건 (서비스별 분류)
- 미팅: 목록
- 기타: ...

---
> 총 근무: N일, 평균 N시간/일, 총 N커밋
```
